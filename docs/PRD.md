## 1. 产品概述

### 1.1 产品定位

**PsyTwin Sentinel** 是新一代校园心理健康全域决策与管理平台，旨在通过数字孪生技术实现学生心理健康的「无感筛查 — 智能预警 — 精准干预 — 康复闭环」全链路管理。

### 1.2 核心价值

| 价值主张 | 说明 |
|----------|------|
| **非侵入式检测** | 摒弃传统问卷，通过 VR 交互与生理传感器实现「无感筛查」 |
| **多模态融合** | 整合语音情感、微表情、行为动作，心率皮电等多维度数据 |
| **AI 增强决策** | Qwen 大模型提供个性化干预建议，RAG 知识库引用专业文献 |
| **闭环联动** | 小程序日常监测 + VR 深度干预 + Web 决策后台，三端数据互通 |

### 1.3 目标用户

| 用户角色 | 使用场景 | 核心需求 |
|----------|----------|----------|
| **学校管理层** | 全域态势看板 | 宏观了解全校心理健康态势，识别高风险群体 |
| **心理咨询师** | 风险处理、咨询记录 | 查看预警详情，处理干预工单、记录咨询过程 |
| **辅导员** | 学生关注、初步干预 | 关注负责班级学生，及时响应预警、参与干预配合 |
| **系统管理员** | 设备管理，系统配置 | 管理 VR 设备、配置预警阈值，维护知识库 |

## 3. 功能模块详细说明

### 3.1 M01：全域态势指挥中心

**产品定位**：学校层面的宏观决策看板，实现心理健康状态的「一屏通览」。

#### 功能清单

| 功能点 | 描述 | 优先级 |
|--------|------|--------|
| F01.1 关键指标统计 | 展示在校学生总数、活跃预警数、进行中干预任务、AI 准确率 | P0 |
| F01.2 心理热力分布图 | 通过散点图模拟校区地图，展示各院系心理压力指数分布 | P0 |
| F01.3 实时预警雷达 | 实时推送最新学生异常预警（心率激增、语音颤抖等） | P0 |
| F01.4 干预转化率漏斗 | 展示从「发现风险」到「康复闭环」的全流程转化漏斗 | P1 |

#### 关键指标卡片 (StatCards)

| 指标 | 说明 |
|------|------|
| 在校学生 | 接入学校教务系统的在校学生总数 |
| 活跃预警 | 当前待处理的高风险预警数量 |
| 干预中 | 正在进行干预的学生数量 |
| AI 准确率 | AI 预警准确率（人工确认正确的比例） |

#### 交互设计

- 点击热力图区域 → 跳转至对应院系的学生列表
- 点击预警雷达 → 跳转至风险溯源中心
- 点击漏斗阶段 → 筛选对应状态的工单

**产品定位**：咨询师处理高风险个体的核心业务区，实现「发现问题 — 核验数据 — 决策干预」的闭环。

#### 功能清单

| 功能点 | 描述 | 优先级 |
|--------|------|--------|

| F02.2 多模态证据链 | 查看 VR 交互中的语音情感波形、生理特征曲线 | P0 |
| F02.3 AI 风险评估报告 | Qwen 模型生成的深度分析报告（语音、生理、行为三维度） | P0 |
| F02.4 决策操作 | 「确认干预」和「解除预警」两个操作入口 | P0 |
| F02.5 工单状态流转 | 预警状态：待处理 → 处理中 → 已闭环 | P1 |

#### 多模态数据展示

| 数据类型 | 可视化形式 | 数据来源 |
|----------|------------|----------|
| 语音情感 | 波形图 + 情感标签 | VR 设备麦克风 |
| 心率变化 | 折线图 + 异常标记 | 生理监测手环 |
| 行为动作 | 时序事件流 | VR 设备动作捕捉 |
| 微表情 | 特征点分布热图 | VR 设备眼动追踪 |

#### AI 风险评估报告结构

```
## 风险评估报告

### 语音维度
- 情感倾向：[焦虑/平静/紧张/低落]
- 语音颤抖指数：[0.0-1.0]
- 结论：[基于语音情感分析的风险判断]

### 生理维度
- 心率变异：[HRV数值]
- 皮电反应：[GSR数值]
- 结论：[基于生理指标的风险判断]

### 行为维度
- 动作强度：[0.0-1.0]
- 回避次数：[数值]
- 结论：[基于行为分析的风险判断]

### 综合风险等级
- [红色/橙色/黄色]
```

## 4. 用户交互与流程

### 4.1 典型用户流程

#### 流程一：风险预警处理

```
1. [系统] VR 设备/小程序采集多模态数据
       ↓
2. [AI] Qwen 模型计算风险分数，超过阈值则生成预警
       ↓
3. [全域态势] 预警雷达显示新预警，状态变为「待处理」
       ↓
4. [风险溯源] 咨询师查看预警详情、多模态证据
       ↓
5. [AI] 生成风险评估报告
       ↓
6. [咨询师] 确认干预 or 解除预警
       ↓
7. [干预记录] 创建干预工单 → 分配咨询师
       ↓
8. [咨询] 执行咨询干预，记录咨询内容
       ↓
9. [复查] 干预后再次检测，风险降级 → 工单结案
```

#### 流程二：学生档案查询

```
1. [全域态势] 点击热力图区域或学生数量
       ↓
2. [学生档案] 展示该院系/班级学生列表
       ↓
3. [点击学生] 查看学生心理画像、时间轴
       ↓
4. [干预记录] 查看历史干预记录
       ↓
5. [风险溯源] 如需处理当前预警，直接跳转
```
