# OpenClaw 集成至 AI 配置页面产品需求文档

> **所属项目**：PsyTwin Sentinel（心理孪生·哨兵）
> **文档版本**：v0.1
> **创建日期**：2026-03-13
> **状态**：规划中
> **目标范围**：仅定义 OpenClaw Office 集成到 Sentinel `AI配置` 模块的产品方案、信息架构与 UI/UX 改造要求

---

## 1. 背景与目标

### 1.1 背景

当前 PsyTwin Sentinel 的 **AI 配置** 页面已具备以下基础能力：

- RAG 向量知识库文档上传与索引
- RAG 检索测试
- Prompt 角色预设与系统提示词编辑
- AI 服务连接状态检查

隔壁项目 **OpenClaw Office** 是一个基于 Next.js 构建的多智能体工作流可视化仪表盘，具备：

- 智能体办公室场景可视化
- 请求流转追踪（Request Pipeline）
- 智能体状态与思维展示
- 成本、统计、安全、数据库等运行视图

但 OpenClaw Office 当前采用明显的赛博朋克视觉风格，不符合 PsyTwin Sentinel 面向学校管理后台的专业、克制、中性设计语言，因此不能直接照搬，必须进行信息架构重组与视觉重设计。

### 1.2 集成目标

本次集成的核心目标如下：

1. 将 OpenClaw Office 的多智能体编排能力纳入 Sentinel 的 **AI 配置** 模块下统一管理。
2. 将原 OpenClaw 的赛博风格改造为符合 Sentinel 当前设计体系的后台管理 UI。
3. 为 AI 配置建立清晰的 **次级导航结构**，避免单页信息堆叠。
4. 将 OpenClaw 原有核心页面能力保留并重新组织为中文产品结构。
5. 为后续代码集成、路由拆分、组件迁移与 API 对接提供统一 PRD 依据。

### 1.3 非目标

本阶段 **不包含** 以下内容：

- 不直接实施 OpenClaw 前端组件迁移
- 不直接改造 OpenClaw API 或新建完整后端服务
- 不在本 PRD 中定义微信小程序侧任何页面或逻辑
- 不新增与 AI 配置无关的一级导航模块

## 2. 现状分析

### 2.1 Sentinel 当前 AI 配置现状

当前路由与实现：

- 路由：`/ai-config`
- 页面入口：`app/(dashboard)/ai-config/page.tsx`
- 视图组件：`components/views/ai-config-view.tsx`

当前页面结构为左右双栏：

- 左侧：RAG 知识库管理、文档上传、文档列表、检索测试
- 右侧：Prompt 预设与系统提示词配置

当前页面问题：

1. AI 配置承载能力不足，难以容纳 OpenClaw 集成后的多视图内容。
2. RAG、Prompt、工作流编排、模型状态混在同一层，不利于模块化扩展。
3. 缺乏统一的 AI 能力导航入口，不利于后续维护。

### 2.2 Sentinel 当前导航可复用模式

`components/dashboard-sidebar.tsx` 中，**系统设置** 已实现较成熟的次级导航模式：

- 一级菜单：系统设置
- 二级菜单：基础设置 / 用户管理 / 数据同步 / 安全策略 / 通知管理
- 切换机制：`?tab=` 查询参数驱动

该模式已被验证适合本项目，可直接作为 AI 配置升级为次级导航的参考范式。

### 2.3 OpenClaw Office 当前结构

OpenClaw Office 当前主页面采用单页 Tab 结构，核心 Tab 包括：

- `office`：办公室视图（核心可视化）
- `stats`：统计数据
- `team`：智能体状态 / 思维展示
- `cost`：成本节省
- `security`：安全视图
- `database`：数据库状态

核心组件包括：

- `GridOffice` / `IsometricOffice`
- `RequestPipeline`
- `ActivityLog`
- `StatsCards`
- `TeamDashboard`
- `CostDashboard`
- `SecurityDashboard`
- `DatabaseDashboard`

### 2.4 OpenClaw 当前 UI 风格问题

OpenClaw 当前样式中存在以下不适配 Sentinel 的元素：

- 霓虹紫 / 霓虹青 / 亮粉色主色系统
- `.cyber-grid` 背景网格
- `.scanlines` 扫描线
- `.glitch` 故障动效
- `.cyber-rain` 背景雨幕
- `Orbitron` / `JetBrains Mono` 强赛博字体
- 高发光边框与玻璃拟态大面积叠加

这些风格不适合校园心理健康管理后台，会削弱平台的专业感、可信感和稳定感。

## 3. 集成原则

### 3.1 信息架构原则

1. **统一收口**：OpenClaw 能力全部收口到现有 `AI配置` 一级导航下，不新增新的主导航组。
2. **次级导航化**：将 AI 配置拆为多个次级导航，替代现有“大而全单页”。
3. **能力分层**：将“编排可视化”“知识增强”“模型策略”三类能力明确区隔。
4. **中文化呈现**：页面标题、模块名称、操作文案、状态提示统一使用中文。

### 3.2 视觉设计原则

1. 采用 Sentinel 既有后台风格：`Card + CardHeader + CardContent` 为主。
2. 使用本项目现有中性色、主色、状态色，不保留 OpenClaw 霓虹配色。
3. 保留“实时感”和“流程感”，但弱化炫技动效，提升信息可读性。
4. 使用中文标题、中文指标名、中文状态文案。
5. 动效应服务于信息理解，不得喧宾夺主。

### 3.3 技术与集成原则

1. 优先复用 Sentinel 已有导航模式（侧边栏 + `?tab=`）。
2. 优先迁移 OpenClaw 的业务逻辑与视图结构，不迁移其整套全局样式体系。
3. OpenClaw 相关组件在 Sentinel 内必须被重新封装为符合本项目目录与命名规范的组件。
4. 所有新增命名、文案、状态定义应优先面向校园心理平台的实际语义，而非保留通用工程化黑客术语。

## 4. 新的信息架构方案

## 4.1 一级导航保持不变

- 系统配置
  - AI配置
  - 系统设置

### 4.2 AI 配置次级导航方案

AI 配置升级为带二级菜单的配置中心，建议拆分为以下 3 个次级导航：

| 次级导航 | 定位 | 说明 |
|---|---|---|
| OpenClaw 编排 | 多智能体工作流与运行态可视化 | 承接 OpenClaw Office 的主要能力 |
| RAG 向量知识库 | 知识接入、文档索引、检索测试 | 承接现有 AI 配置页左侧能力 |
| 模型与策略中心 | Prompt、模型参数、调用策略、运行状态 | 承接现有 AI 配置页右侧能力，并为后续扩展预留空间 |

### 4.3 路由建议

延续本项目现有模式，优先使用：

- `/ai-config?tab=openclaw`
- `/ai-config?tab=rag`
- `/ai-config?tab=strategy`

这样可保持：

- 与系统设置导航模式一致
- 侧边栏高亮逻辑可复用
- 不引入过早的深层嵌套路由复杂度

## 5. OpenClaw 页面映射方案

### 5.1 原页面能力到新结构的映射

| OpenClaw 原 Tab / 模块 | Sentinel 集成后归属 | 中文名称建议 | 说明 |
|---|---|---|---|
| office | OpenClaw 编排 | 智能体编排总览 | 核心办公室/网格编排视图 |
| RequestPipeline | OpenClaw 编排 | 请求流转追踪 | 展示请求生命周期与分派过程 |
| ActivityLog | OpenClaw 编排 | 运行活动日志 | 展示实时事件流 |
| stats | OpenClaw 编排 | 编排统计看板 | 展示任务量、交互量、处理效率 |
| team | OpenClaw 编排 | 智能体状态面板 | 展示智能体分工、状态、思维摘要 |
| cost | OpenClaw 编排 | 资源消耗分析 | 展示 Token / 成本 / 节省估算 |
| security | OpenClaw 编排 | 编排安全审计 | 展示权限、安全事件、调用审计 |
| database | OpenClaw 编排 | 数据存储状态 | 展示数据库、事件缓存、状态健康度 |

### 5.2 页面组织建议

在 `OpenClaw 编排` 次级导航下，不建议继续暴露 OpenClaw 式英文顶层 Tab，而建议采用 **页面内分组或三级标签** 组织：

#### 方案 A：单页分区式（推荐）

`OpenClaw 编排` 页面内按纵向版块组织：

1. 顶部概览区：在线状态、运行摘要、核心指标
2. 主体左侧：智能体编排主视图
3. 主体右侧：请求流转追踪 + 活动日志
4. 下方扩展区：统计、智能体状态、资源消耗、安全、数据库状态

适用原因：

- 更接近 Sentinel 当前后台页面风格
- 减少层层切换，便于统一管理体验
- 适合管理后台高信息密度场景

#### 方案 B：页面内三级 Tab（备选）

将 `OpenClaw 编排` 再拆为：

- 总览
- 统计分析
- 智能体状态
- 资源消耗
- 安全审计
- 数据状态

若后期模块内容持续膨胀，可考虑该方案。

## 6. 各次级导航详细需求

### 6.1 次级导航一：OpenClaw 编排

#### 产品定位

作为平台的 **多智能体运行态指挥台**，面向管理员展示 AI 编排系统的执行链路、智能体分工、运行状态与资源开销。

#### 功能清单

| 功能点 | 描述 | 优先级 |
|---|---|---|
| F01 智能体编排总览 | 可视化展示智能体在办公室/工作区中的分布与工作状态 | P0 |
| F02 请求流转追踪 | 展示任务从接收、分析、创建、分派、执行到完成的过程 | P0 |
| F03 运行活动日志 | 展示实时日志、关键动作、状态变更 | P0 |
| F04 编排统计看板 | 展示任务总量、处理量、响应效率、趋势 | P1 |
| F05 智能体状态面板 | 展示各角色当前状态、分工、工作负载 | P1 |
| F06 资源消耗分析 | 展示 Token 消耗、成本估算、效率收益 | P1 |
| F07 编排安全审计 | 展示调用安全、权限校验、异常事件 | P2 |
| F08 数据存储状态 | 展示数据库 / 缓存 / 事件总线状态 | P2 |

#### 页面中文文案规范

- Main Office → 智能体编排总览
- Interaction Stats → 编排统计看板
- Agent Thoughts → 智能体状态面板
- Cost Savings → 资源消耗分析
- Security → 编排安全审计
- Database → 数据存储状态
- Request Pipeline → 请求流转追踪
- Activity Log → 运行活动日志
- LIVE → 实时运行中

### 6.2 次级导航二：RAG 向量知识库

#### 产品定位

作为平台的 **知识增强管理中心**，用于维护心理干预、危机应对、咨询伦理等专业知识文档，并支持检索验证。

#### 功能范围

沿用并整理当前已有能力：

- 文档上传
- 文档列表
- 文档索引状态
- 文档删除
- RAG 检索测试

#### 改造要求

1. 保留现有业务能力，提升结构清晰度。
2. 在标题与说明文案层面统一中文化、产品化。
3. 明确知识库适用场景，例如危机干预、CBT 手册、校园流程规范等。

### 6.3 次级导航三：模型与策略中心

#### 产品定位

作为平台 AI 的 **策略配置中枢**，统一管理 Prompt、模型参数、角色模板、调用策略与运行状态。

#### 功能清单

| 功能点 | 描述 | 优先级 |
|---|---|---|
| F11 Prompt 模板管理 | 编辑系统提示词与场景策略模板 | P0 |
| F12 角色预设管理 | 管理危机干预专家、情感疏导师等角色模板 | P0 |
| F13 模型连接状态 | 展示 AI 服务是否可用 | P1 |
| F14 调用策略配置 | 配置温度、回复长度、路由策略、兜底策略 | P1 |
| F15 服务重载与测试 | 提供模型重载、快速验证入口 | P1 |
| F16 策略审计预留 | 为未来记录策略变更历史预留区块 | P2 |

#### 命名理由

不使用“Prompt 配置”作为第三个导航标题，是因为后续集成后这里会承载的不只是文本模板，而是更高一层的平台级 AI 策略能力，因此“模型与策略中心”更稳妥、可扩展。

## 7. UI / UX 改造方案

### 7.1 总体设计方向

将 OpenClaw 从“赛博朋克沉浸式演示看板”改造成“专业、可信、克制的校园心理平台 AI 运维配置台”。

### 7.2 必须移除的视觉元素

- 深黑霓虹背景
- 发光描边与重阴影霓虹边框
- 扫描线特效
- 故障闪烁（glitch）效果
- 黑客雨幕 / cyber-rain
- 强游戏化 emoji 主表达方式
- 赛博字体作为主标题字体

### 7.3 替换后的视觉规范

| 维度 | OpenClaw 当前风格 | Sentinel 集成后风格 |
|---|---|---|
| 背景 | 深黑赛博底色 | 本项目默认页面背景/卡片背景 |
| 主色 | 霓虹青 / 紫 / 粉 | Sentinel 主色 + 中性色 |
| 字体 | Orbitron / JetBrains Mono | 项目默认中文字体体系 |
| 卡片 | 玻璃拟态 + 强发光 | 标准 Card + 弱阴影 |
| 状态表达 | 霓虹/emoji 强提示 | Badge、图标、状态文本 |
| 动效 | 强炫光、飞线、glitch | 轻量过渡与状态动效 |

### 7.4 交互体验要求

1. 关键指标优先可读性，不追求视觉噱头。
2. 流程可视化应帮助管理员理解“谁在处理什么”，而不是制造炫酷感。
3. 所有操作按钮、说明文本、状态提示统一使用中文。
4. 动态区域需兼顾空状态、加载中、异常状态三类反馈。

- 页面标题：OpenClaw 编排

#### 中部主体

- 左 2/3：智能体编排主画布

- 智能体状态面板
- 资源消耗分析

- 数据存储状态

- 上部：知识库介绍与上传入口

- 上部：AI 服务状态 + 快捷操作

## 9. 实施建议

### 9.1 建议实施阶段

#### 第一阶段：信息架构重构

- 将 AI 配置升级为次级导航模式

2. 请求流转追踪
3. 活动日志
4. 智能体状态面板
5. 统计 / 成本 / 安全 / 数据状态

## 11. 结论

OpenClaw Office 应作为 **PsyTwin Sentinel 的 AI 编排能力层** 集成进 `AI配置`，而不是作为独立风格的外来页面挂载。最佳方案是：

- 保持 AI 配置为一级入口
- 拆分为 `OpenClaw 编排 / RAG 向量知识库 / 模型与策略中心` 三个次级导航
- 在视觉上全面脱离赛博风，转化为符合校园心理管理后台的专业界面
- 在结构上保留 OpenClaw 的核心能力与运行态可视化价值

该方案既能最大化复用 OpenClaw 的成熟能力，又能保证 Sentinel 在产品体验、品牌一致性与维护性上的统一。
